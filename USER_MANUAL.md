## システム概要

### このシステムでできること

WannaVレッスン分析システムは、Google Meetのレッスン録画を自動的に分析し、講師のパフォーマンスを評価するシステムです。

**主な機能**:
- 📊 **月次評価**: 毎月末に全講師からランダムに2レッスンずつ自動評価
- 📈 **グラフ表示**: 講師発話比率とレッスン数の推移を可視化
- 🎥 **詳細分析**: 個別レッスンの詳細なKPI・感情分析
- 👥 **講師フィルター**: 特定の講師のデータだけを表示

### 自動評価のタイミング

- **毎月末日の23:00（日本時間）**に自動実行
- 全講師の今月の録画から**ランダムに2件ずつ**選んで分析
- 翌朝にはダッシュボードで結果を確認できます

### 画面の見方

#### 1. 講師選択エリア

```
┌─────────────────────────────────┐
│ 講師を選択: [全講師 ▼]           │
└─────────────────────────────────┘
```

- **全講師**: すべての講師のデータを表示
- **特定の講師**: 選択した講師のデータのみ表示

**使い方**:
1. ドロップダウンをクリック
2. 見たい講師を選択
3. グラフとテーブルが自動更新されます

---

#### 2. 月次評価エリア

##### 📈 講師発話比率 推移グラフ

```
講師発話比率 推移
100% ┤
 80% ┤     ●──●
 60% ┤  ●─┘     
 40% ┤
  0% ┼─────────────────
     2025-12  2026-01  2026-02
```

**見方**:
- **縦軸**: 講師の発話比率（%）
- **横軸**: 月
- **理想値**: 50-60%（生徒がしっかり話せている）
- **注意値**: 70%以上（講師が話しすぎ）

**色の意味**:
- 🟢 **緑（60%以下）**: 良好
- 🟡 **黄色（60-70%）**: 普通
- 🔴 **赤（70%以上）**: 要改善

---

##### 📊 レッスン数推移グラフ

```
レッスン数推移
 3 ┤     ██
 2 ┤  ██ ██ ██
 1 ┤
 0 ┼─────────────────
   2025-12  2026-01  2026-02
```

**見方**:
- 各月に評価されたレッスン数（通常2件）
- データが少ない月は信頼性が下がります

---

##### 📋 月次サマリーテーブル

| 月 | 講師 | レッスン数 | 平均発話比率 | 平均沈黙回数 | 合計時間 |
|----|------|-----------|------------|-------------|---------|
| 2026-01 | きょうへい先生 | 2 | 52.4% | 18.5 | 112分 |
| 2026-01 | あやこ先生 | 2 | 65.3% | 12.0 | 98分 |

**各列の意味**:

- **月**: 評価対象の月
- **講師**: 講師名
- **レッスン数**: その月に評価したレッスン数
- **平均発話比率**: 講師が話している時間の割合
  - 🟢 60%以下: 良好
  - 🟡 60-70%: 普通
  - 🔴 70%以上: 要改善
- **平均沈黙回数**: 15秒以上の沈黙が発生した回数
  - 少ないほど良い（目安: 20回以下）
- **合計時間**: 評価した全レッスンの合計時間

##### レッスン詳細カード

```
┌────────────────────────────────────────┐
│ WannaVレッスン予約 (山田太郎)           │
│ 2026-01 | きょうへい先生 | 61分16秒    │
├────────────────────────────────────────┤
│ ┌─────────┐ ┌─────────┐               │
│ │講師発話  │ │生徒発話  │               │
│ │ 52.4%   │ │ 47.6%   │               │
│ └─────────┘ └─────────┘               │
│ ┌─────────┐ ┌─────────┐               │
│ │最長モノ  │ │生徒ター  │               │
│ │ローグ    │ │ン数     │               │
│ │  1分    │ │ 601回   │               │
│ └─────────┘ └─────────┘               │
└────────────────────────────────────────┘
```

**主要KPI**:

1. **講師発話比率**: 講師が話している時間の割合
   - 理想: 50-60%
   
2. **生徒発話比率**: 生徒が話している時間の割合
   - 理想: 40-50%

3. **最長モノローグ**: 講師の最長連続発話時間
   - 理想: 3分以内
   - 注意: 5分以上

4. **生徒ターン数**: 生徒が発言した回数
   - 多いほど良い（インタラクティブ）

##### 感情シグナル

```
┌────────────────────────────────────────┐
│ 感情シグナル                            │
├────────────────────────────────────────┤
│ ┌──────┐ ┌──────┐ ┌──────┐           │
│ │困惑  │ │ストレス│ │ポジティブ│         │
│ │30.0% │ │20.0% │ │30.0% │           │
│ └──────┘ └──────┘ └──────┘           │
└────────────────────────────────────────┘
```

**見方**:
- **困惑**: 生徒が理解できていない可能性
- **ストレス**: 生徒がプレッシャーを感じている
- **ポジティブ**: 生徒が楽しんでいる

各セクションには具体的なタイムスタンプ付きで、どの場面で発生したか表示されます。

---

##### 改善アドバイス

AIが自動生成した具体的な改善提案が表示されます。

**例**:
```
発話比率は良好ですが、生徒の困惑シグナルが多く見られます。
特に、00:45と19:34で生徒が理解に苦しんでいる様子が見られました。
説明の後に「分かった?」などの確認を入れることをお勧めします。
```

---

##### 推奨アクション

具体的な次のアクションが番号付きリストで表示されます。

**例**:
```
1. 説明後に理解度確認を行う（「分かった?」「質問ある?」）
2. 生徒の反応を見て、説明のペースを調整する
3. 3分以上話し続けないよう意識する
```

## 月次評価レポートの見方

### Google Sheets で確認する

**スプレッドシートURL**: [こちら](https://docs.google.com/spreadsheets/d/1gFrIbkRxNcpKuT0vRNfaUdSrJWynlCdfqhGQz9vWwWo/edit)

#### シート構成

1. **monthly_lessons**: 各レッスンの詳細データ
2. **monthly_tutors**: 講師ごとの月次サマリー

---

### monthly_lessons シート

各レッスンの詳細なKPIが記録されています。

**主要カラム**:
- `date_jst`: 評価対象の月（例: 2026-01）
- `tutor_name`: 講師名
- `file_name`: 動画ファイル名（生徒名含む）
- `duration_sec`: レッスン時間（秒）
- `talk_ratio_tutor`: 講師発話比率（0.524 = 52.4%）
- `student_silence_over_15s_count`: 15秒以上の沈黙回数
- `improvement_advice`: 改善アドバイス
- `status`: 処理ステータス（OK / ERROR）

---

### monthly_tutors シート

講師ごとの月次サマリーが記録されています。

**カラム**:
- `date_jst`: 月（例: 2026-01）
- `tutor_name`: 講師名
- `lessons_count`: 評価したレッスン数（通常2件）
- `avg_tutor_talk_ratio`: 平均講師発話比率
- `avg_silence_15s_count`: 平均沈黙回数
- `total_duration_min`: 合計時間（分）

**確認のポイント**:
1. `lessons_count` が 2 になっているか
2. `avg_tutor_talk_ratio` が 0.5-0.6（50-60%）の範囲内か
3. `avg_silence_15s_count` が 20 以下か

## よくある質問

### Q1. データはいつ更新されますか？

**A**: 毎月末日の23:00（日本時間）に自動実行され、翌朝にはダッシュボードで確認できます。

**例**:
- 2026年1月31日 23:00: 1月のデータを自動評価
- 2026年2月1日 朝: ダッシュボードで結果確認可能

---

### Q2. なぜ全レッスンではなくランダム2件だけ？

**A**: 以下の理由があります：

1. **コスト削減**: 全レッスンを分析すると月間約1,050件で高コスト
2. **十分な信頼性**: 統計的に2件あれば傾向は把握できる
3. **公平性**: ランダム選択なので特定のレッスンに偏らない

---

### Q3. データが表示されない場合は？

**チェックポイント**:

1. **月次ジョブが実行されたか確認**
   - Google Sheets で `monthly_lessons` シートにデータがあるか

2. **講師フィルターを確認**
   - 「全講師」に戻してみる

3. **ブラウザをリフレッシュ**
   - F5キーまたは更新ボタンを押す

4. **それでも表示されない場合**
   - システム管理者に連絡してください

---

### Q4. 講師発話比率の目安は？

**理想的な範囲**: 50-60%

**評価基準**:
- 🟢 **40-60%**: 優良（生徒が十分に話せている）
- 🟡 **60-70%**: 普通（改善の余地あり）
- 🔴 **70%以上**: 要改善（講師が話しすぎ）
- 🟡 **40%未満**: 普通（生徒が話しすぎ、または沈黙が多い）

---

### Q5. 感情シグナルの信頼性は？

**A**: AIによる推定値なので、参考程度にご利用ください。

**精度を上げるために**:
- タイムスタンプを確認して、実際の録画を見直す
- 複数レッスンの傾向を見る
- 生徒のフィードバックと照らし合わせる

---

### Q6. 特定の月のデータだけ見たい

**A**: 現在、ダッシュボードは全期間のデータを表示します。

**代替方法**:
1. Google Sheets で該当月をフィルタリング
2. 講師フィルターと組み合わせて使う

---

### Q7. 過去のデータは保存されますか？

**A**: はい、Google Sheets にすべて保存されます。

- **monthly_lessons**: 各レッスンの詳細
- **monthly_tutors**: 月次サマリー

データは削除されない限り、永続的に保存されます。

---

### Q8. エラーが表示される場合は？

**よくあるエラー**:

1. **「データがありません」**
   - 月次ジョブが未実行
   - 該当月にレッスンがない

2. **グラフが表示されない**
   - ブラウザのキャッシュをクリア
   - 別のブラウザで試す

3. **レッスン詳細が見れない**
   - そのレッスンが ERROR ステータスの可能性
   - Google Sheets で確認

---

### Q9. 改善アドバイスはどう活用する？

**推奨される使い方**:

1. **個別フィードバック**
   - 各講師に月次レポートとして共有
   - 具体的な改善点を一緒に確認

2. **研修材料**
   - 共通の課題を抽出
   - グループ研修のテーマに

3. **経過観察**
   - 月ごとの推移を確認
   - 改善が見られたら褒める

---
